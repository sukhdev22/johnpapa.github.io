<p>I <a href="http://www.jetbrains.com/resharper/beta/beta.html">downloaded Resharper 4 beta</a> today. I've been installing several versions of the <a href="http://www.jetbrains.com/resharper/beta/beta.html">Resharper</a> 4 EAP as they have been released. Most have been pretty stable over the past month, which makes it no surprise that they released a beta. Yes its a later than most, including myself, expected to see a new version that supports all of the features of .NET 3.5 and Visual Studio 2008 (like LINQ, lambdas, and extension methods) but the EAPs have kept me going. Frankly I have gotten addicted to refactoring tools. <a href="http://www.jetbrains.com/resharper/beta/beta.html">Resharper</a> has less eye candy that CodeRush and Refactor Pro, but its more efficient for me.</p> <p>I'd love to see Developer Express really step up and compete with Resharper on their code analysis and other great aspects. There are some great features from both. But I digress ...</p> <p>The beta version of Resharper 4 is out, and it supports LINQ and all of the other cool new toys. One thing I love about Resharper is the renaming feature and how it works with XAML. This has been awesome while writing my latest Silverlight projects.</p> <p>Using the SHIFT - F6 key combination you can rename a variable, class, etc. But you can also do this in XAML and have it update both the XAML and the .NET code in the code behind. Using this shortcut on the Person class reference, Resharper first creates the Person class for me. Then I use ALT-ENTER on the property name for City and it gives me this option to create the property:</p>

<p>It will create automatic properties for you or you can have it create regular property getters and setters. For me, I use a live templates to create my field and property combinations.</p>
